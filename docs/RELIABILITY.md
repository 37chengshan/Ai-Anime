# 可靠性规范

## 服务可用性目标

| 服务 | SLA | 度量 |
|------|-----|------|
| Web | 99.9% | 响应时间 < 500ms |
| API | 99.9% | 响应时间 < 200ms |
| Worker | 99.5% | 任务处理时间 < 30s |

## 错误处理

### 前端

- 所有 API 调用必须有错误处理
- 用户友好错误信息
- 网络错误自动重试（最多 3 次）
- 关键操作有加载状态

### 后端

- 所有服务层返回结构化错误
- 日志包含完整错误上下文
- 区分客户端错误（4xx）和服务端错误（5xx）
- 关键路径有熔断机制

### Worker

- 任务失败自动重试（指数退避）
- 重试上限 5 次
- 死信队列处理最终失败的任务

## 监控与告警

### 指标

- 请求量（QPS）
- 错误率
- 响应时间（P50, P95, P99）
- 队列长度

### 告警阈值

| 指标 | 阈值 | 级别 |
|------|------|------|
| 错误率 | > 5% | Warning |
| 错误率 | > 10% | Critical |
| 响应时间 P95 | > 1s | Warning |
| 响应时间 P95 | > 3s | Critical |
| 队列长度 | > 1000 | Warning |

## 灾难恢复

### 数据备份

- PostgreSQL：每日全量备份，WAL 归档
- Redis：RDB 每小时快照

### 恢复时间目标

- RTO（恢复时间目标）：< 1 小时
- RPO（恢复点目标）：< 15 分钟
