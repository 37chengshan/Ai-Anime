# Figmake AI 前端生成提示词文档

> 用于指导 AI 生成前端界面的功能描述文档
> 重点聚焦功能实现,不限定具体主题和内容

---

## 一、项目定位

**AI 动漫/漫画创作社区平台** — 创作者分享 AI 生成的动漫漫画作品、交流创作技巧的平台,支持付费订阅获取专业教程和资源。

### 核心用户
- **创作者**: AI 动漫/漫画创作者
- **学习者**: 想提升 AI 创作技巧的用户
- **付费会员**: 订阅教程和资源的用户

---

## 二、核心功能模块

### 2.1 内容展示系统

#### 作品展示区
- **瀑布流布局**: Pinterest/小红书风格的作品卡片展示
- **作品卡片组件**:
  - 缩略图预览(自适应尺寸)
  - 作者信息头像+昵称
  - 作品标题(限制字数)
  - 创建时间/更新时间
  - 点赞数、评论数、浏览量统计
  - 快捷操作按钮(点赞、收藏、分享)
- **筛选与排序**:
  - 分类筛选(动漫、漫画、插画、概念图)
  - 时间排序(最新、热门、精选)
  - 搜索功能(标题、作者、标签)
- **无限滚动加载**: 滚动到底部自动加载更多内容

#### 作品详情页
- **大图/多图展示**: 支持单图或多图作品展示
- **作品信息卡片**:
  - 作者信息(头像、昵称、简介、关注按钮)
  - 作品标题和描述
  - 创建日期和更新日期
  - 使用工具/模型信息(可选显示)
  - 标签系统(点击可跳转相关作品)
- **社交功能**:
  - 点赞按钮(动画效果)
  - 收藏按钮(添加到个人收藏夹)
  - 分享按钮(复制链接、分享到社交媒体)
  - 评论区域(见评论区组件)
- **相关推荐**:
  - 同作者其他作品
  - 相似风格作品推荐
  - 标签相关作品

## 三、UI/UX 设计规范

### 3.1 设计风格
- **风格**: 极简现代风(参考小红书、Figma)
- **配色**: 橙色/暖色系为主色调
- **字体**: 无衬线字体,清晰易读
- **间距**: 合理留白,呼吸感强

### 3.2 响应式设计
- **桌面端**: 多列布局,宽屏优化
- **平板端**: 两列布局,自适应调整
- **移动端**: 单列布局,底部导航

### 3.3 交互设计
- **微交互**:
  - 点赞动画(心形动画)
  - 悬停效果(卡片阴影变化)
  - 加载动画(骨架屏)
- **反馈机制**:
  - 操作成功提示(Toast)
  - 错误提示(Alert)
  - 确认对话框(二次确认)
- **性能优化**:
  - 懒加载图片
  - 无限滚动(分页加载)
  - 骨架屏占位

## 四、技术架构要求

### 4.1 技术栈
- **框架**: Next.js 15+ App Router
- **数据库**: PostgreSQL (Supabase/Neon)
- **认证**: Clerk
- **支付**: Stripe 订阅
- **AI**: 兼容 OpenAI 格式 API
- **图片存储**: Vercel Blob / Cloudflare R2
- **UI**: Tailwind CSS

### 4.2 性能要求
- **加载速度**: 首屏加载 < 2s
- **响应时间**: API 响应 < 300ms
- **SEO**: 基础 SEO 优化(标题、描述、结构化数据)

### 4.3 安全要求
- **认证安全**: Clerk 安全认证
- **支付安全**: Stripe 安全支付
- **数据安全**: 用户数据加密存储
- **权限控制**: 会员权限验证

## 五、Agent-Native 功能集成

### 5.1 工具化设计
将所有用户操作封装为原子性工具:
- `read_works`: 读取作品列表
- `create_work`: 创建新作品
- `update_work`: 更新作品信息
- `delete_work`: 删除作品
- `like_work`: 点赞作品
- `comment_work`: 评论作品
- `follow_user`: 关注用户
- `search_content`: 搜索内容
- `subscribe_membership`: 订阅会员

### 5.2 AI Agent 交互
- **自然语言指令**: 用户通过 Chat 发送指令
- **工具组合**: Agent 自动组合多个工具完成复杂任务
- **状态反馈**: 实时展示执行进度和结果
- **错误恢复**: Agent 自动处理错误和重试

### 5.3 Parity 原则
确保 UI 能做的所有操作,AI Agent 都能通过工具完成:
- UI 发布作品 = Agent 调用 create_work 工具
- UI 点赞评论 = Agent 调用 like_work/comment_work 工具
- UI 搜索内容 = Agent 调用 search_content 工具

## 九、关键功能流程

### 9.1 作品发布流程
```
用户点击上传按钮 → 打开上传界面 → 选择图片文件 → 
填写作品信息(标题、描述、标签) → 选择分类 → 
设置隐私选项 → 点击发布按钮 → 上传进度显示 → 
上传成功提示 → 作品展示在个人主页和首页
```

### 9.2 评论流程
```
用户点击评论按钮 → 打开评论区 → 输入评论内容 → 
可选:使用AI评论助手生成建议 → 点击提交 → 
评论添加到评论列表 → 评论数更新 → 
通知作者(新评论通知)
```

### 9.3 订阅流程
```
用户点击订阅按钮 → 打开订阅页面 → 
查看会员权益 → 选择订阅方案(月度/年度) → 
点击支付按钮 → Stripe 支付流程 → 
支付成功 → 会员状态更新 → 
会员权限解锁 → 显示会员徽章
```

### 9.4 AI Chat 流程
```
用户打开 AI Chat 侧边栏 → 输入指令/问题 → 
Agent 分析指令 → 调用相关工具 → 
执行操作 → 返回结果 → 
结果展示(Markdown渲染) → 可选:继续对话
```
