# 生产事故响应指南

## 事故等级

| 等级 | 定义 | 响应时间 | 通知 |
|------|------|----------|------|
| P0 | 服务完全不可用 | 15 分钟 | 立即通知所有人 |
| P1 | 核心功能不可用 | 30 分钟 | 通知负责人 |
| P2 | 非核心功能不可用 | 2 小时 | 工单记录 |
| P3 | 体验降级 | 1 工作日 | 工单记录 |

## 响应流程

### 1. 发现

- Sentry 告警
- 用户反馈
- 监控异常

### 2. 评估

- 确认影响范围
- 判定事故等级
- 指定事故负责人

### 3. 止血

- 优先恢复服务
- 可接受的临时方案：
  - 回滚部署
  - 禁用问题功能
  - 增加限流
  - 切换到降级模式

### 4. 修复

- 定位根因
- 实施修复
- 验证修复效果

### 5. 复盘

- 记录事故时间线
- 分析根因
- 制定预防措施
- 更新文档和监控

## 常见事故场景

### 数据库连接耗尽

```bash
# 检查连接数
psql -c "SELECT count(*) FROM pg_stat_activity;"

# 终止空闲连接
psql -c "SELECT pg_terminate_backend(pid) FROM pg_stat_activity WHERE state = 'idle' AND query_start < now() - interval '10 minutes';"
```

### Redis 内存满

```bash
# 检查内存使用
redis-cli INFO memory

# 清理过期 key
redis-cli SCAN 0 COUNT 1000
```

### API 响应超时

1. 检查上游服务状态
2. 检查数据库慢查询
3. 检查网络延迟
4. 考虑启用熔断
